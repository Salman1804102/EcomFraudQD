Customer or user reviews poses a great significance for the e-commerce seller and technology owner. In case of the physical good, the manual feedback or survey is required to measure a product’s success in the market. But for the online goods or products, it is no longer a requirements as the consumer themselves can provide their opinion and feedback on purchased goods in online platform. These vast amount of reviews can be very much important for the e-commerce seller if they want to improve the product’s quality, retain the brand impression, customer retention, analyze the revenue earned from products. These reviews can also drive the seller to put immediate attention in emergency cases by looking at what complaints are claiming by consumers. However, analyzing the vast number of these reviews can be very challenging. Apart from that, reviews may contain the variety of emotions, sentiments and complaints. So, manual analysis can be very cumbersome and inefficient as it may take a huge time to manually analyze each reviews and assign them to particular label. Besides, datasets used in existing study poses limitation due to the size or volume of the labeled data. Most of the dataset comprises of 10k-20k labeled data which may cause the concern of generalizing ability. The review can be diverse, there can be numerous diversity based on product category, user’s location, seller type, user’s age etc. Another key limitation of the existing works is that most of the study focused on the sentiment classification or aspect analysis considering the problem as multi-class classification problem, whereas in most of the scenarios specially in long reviews, user usually complain about several issues at a time. Therefore, a review may contain several types of sentiment at a time requiring multi-label complaint dataset.

The google playstore contains millions of applications that have greater number of user reviews. These reviews poses great significance, as analysis of these reviews can potentially improve the application quality if user’s concerns, complaints and issues can be taken into consideration. The app’s review based aspect analysis thus require a great attention as it can help developers to fix issues, bugs and develop new features etc. 

In this study, we propose a zero shot learning based aspect analysis of samsung internet browser’s user reviews. We initially scrapped 2k selective reviews those got rating less than 3 to consider that these reviews are negative reviews and customer are not satisfied with something about the sbrowser. These reviews are analyzed through a zero-shot-learning based pipeline into multiple aspects such as Performance, User Interface, Privacy/Security, Stability/Crashes, Ad Blocking, Extension Support, Compatibility, Download Manager, Sync/Bookmarks, Search Engine, Battery Usage, Voice/Text Input, Dark Mode etc. These aspects are basically classified as multilabel because a review can consist of the complaints about several things, hence we considered all these classes or category to be classified as multilabel. To select the best zsl model we at first  compared the best zsl model using few labled dataset. This step is done to see which zsl model is performing better in classification based on the existing labeled dataset. So, we compared the labeled assign by the zsl model on those labeled dataset in unsupervised manner and then compared the actual label with the zsl assign label to measure accuracy or efficiency of the model. We found ‘Facebook bart-large-mnli (m1)’ model to perform really good and therefore analyzed the sbrowser comments through them. We tested a total of 5 state-of-the-art (Sota) LLM model to find the effectiveness in e-commerce text classification and then selected that model for the for the samsung internet browser comments analysis. The deployed models are Llama, GPT-3.5, DeepSeek, Claude 3.5 Sonnet, and Clause 3.5 Haiku. Among the explored models we found that the DeepSeek outperforms other models while testing with other dataset. 

This analysis will help the samsung to focus on the user feedback, like what they want and what they are complaining about, what features should be included in near future and what steps should be taken immediately that is causing user to restrain them to use sbrowser. It will also help them to compete well with the existing internet browser in the market. It will increase the samsung’s overall brand value, google play store app ranking, user retention, increase advertising revenue and reduce support cause as well. This type of automation in review classification will also help the samsung to focus on immediate action needed. Samsung can employ this type of technique not only on internet browser, it can deploy the equivalent system for its other products as well in online platforms.



In this work our contribution is as follows:

1) We proposed a generalized zero-shot-learning based complaints classification pipeline that can be utilized or adapted in different settings as well with different datasets
2) We compared the performance of various LLM models in existing labeled dataset of aspect based sentiment analysis for a greater of view of their performance on e-commerce text
3) This is the first study to analyze and extracts google playstore based user feedback classification on application product.
4) Analyze the practical chanllenges and limitations of LLM-based aspect analysis in real-world e-commerce settings.
